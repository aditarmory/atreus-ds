Button

Buttons trigger actions. They communicate the action that will happen when a user interacts with them. Atreus buttons use rounded pill-shaped containers to feel approachable and modern.

3 Types · 3 Sizes · 2 States

Filled for primary CTAs, Outline for secondary actions, Ghost for tertiary or low-emphasis actions. All sizes use the same border-radius token: $border-radius-full → 100px.

Usage Guidelines
🎯
Use Filled for primary CTAs only

Reserve the Filled (green) button for the single most important action on a screen. Never use two Filled buttons side by side without hierarchy.

✌️
Outline for secondary actions

Use Outline as a visual counterpart to Filled when you need two buttons side by side — e.g., "Cancel" + "Confirm".

👻
Ghost for tertiary, destructive, or links

Ghost buttons are for lowest-priority actions. In Stacked docked layouts, Ghost sits at the bottom as a clear visual de-emphasis.

📱
Use Docked for mobile bottom actions

Bottom-fixed actions should always use the Docked component, never floating individual buttons. This ensures consistent safe area handling and shadow treatment.

⚡
Use Button Icon for icon-only actions

Never use a regular button with just an icon. The Button Icon component is specifically sized and spaced for icon-only tap targets at 44×44px minimum.

🔢
Limit icons to one side per button

Buttons can have a leading OR trailing icon, not both simultaneously. Leading icons suggest direction or type; trailing icons suggest progression or navigation.

Do / Don't
✓ DO
✓Use Filled + Outline pair for binary choices
✓Full-width Filled in Docked for single primary action
✓Ghost for skip / dismiss actions — no unnecessary weight
✗ DON'T
✗Never use two Filled buttons side by side — no hierarchy
✗Don't use overly long button labels — keep to 1–3 words
✗Don't show disabled buttons without clear context why they're disabled
