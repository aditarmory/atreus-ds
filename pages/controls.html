Usage Guidelines
🔘
Radio for single choice

Use Radio Buttons when the user must pick exactly one option from a mutually exclusive list. Always pre-select the most common or safest default.

☑️
Checkbox for multi-select

Use Checkboxes when the user can pick zero, one, or many options independently. Also valid for a single binary choice like "I agree to the Terms."

🔁
Switch for instant state change

Use Switches when toggling a setting takes effect immediately — no "Save" needed. Don't use a Switch inside a form that requires submission.

🎯
44×44px minimum tap target

All controls must have a 44×44px invisible tap area regardless of the visible component size. This meets WCAG touch target requirements.

👁️
Visible state always

Default, selected, and disabled states must always be visually distinct. Never style a disabled control the same as an enabled one.

⬛
Group logically, label clearly

Related controls belong together visually. Every control needs an adjacent label — never rely on the control icon alone to communicate purpose.

Principles
✨
Visible State

Default, selected, and disabled must always be visually distinct — using border color, fill, and opacity — never relying on color alone.

⬛
Logical Grouping

Group related controls visually. Use consistent alignment and spacing so it's immediately clear which controls belong to a single question.

👍
Sensible Defaults

Pre-select the safest or most common option to reduce cognitive load. Radio Button groups should always have one option selected on load.
